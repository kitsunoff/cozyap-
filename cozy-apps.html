Problem

The Challenge

Current Cozystack model has fundamental scaling limitations.

❌One CRD per application type — Postgres, Redis, Minecraft, WordPress…
❌CRDs are cluster-wide — every new app pollutes the API server
❌Adding a new app requires cluster-admin privileges
❌Tenants cannot create their own application types
Current State
PostgresCluster CRD
RedisCluster CRD
MinecraftServer CRD
WordPressApp CRD
+ N more…
each = cluster-wide resource
Result: Scaling the application catalog is expensive, inflexible, and requires constant cluster-admin involvement.

Solution

Universal Application Abstraction

Two resources replace hundreds of CRDs — regardless of how many apps you support.

Resource Scope Purpose
ApplicationTemplate Cluster How to install, manage and expose an app
Application Namespace A deployed instance of a template
✅ One CRD pair
for ALL application types
✅ Templates reusable
across all tenants
✅ No cluster-admin
to add new apps

Template Example

ApplicationTemplate: Minecraft

apiVersion: apps.cozystack.io/v1alpha1 kind: ApplicationTemplate metadata: name: minecraft spec: displayName: "Minecraft Server" parameters: - name: serverName type: string required: true - name: maxPlayers type: int default: 20 hooks: install: steps: - name: deploy image: alpine/helm:3.14 args: [helm, install, ...] actions: - name: backup displayName: "Create Backup" steps: [...]

Template Author Flow

Creating a New Template

01
Study how the app is normally installed — Helm, manifests, operators
02
Define parameters — what users configure (size, name, replicas, secrets)
03
Write install / upgrade / remove hooks as container steps
04
Add custom actions — backup, restart, config reload…
05
Test in dev environment — apply template, create Application, validate
06
Submit to catalog — available to all tenants immediately
MINIMAL TEMPLATE EXAMPLE
apiVersion: apps.cozystack.io/v1alpha1
kind: ApplicationTemplate
metadata:
  name: my-app
spec:
  parameters:
    - name: replicas
      type: int
      default: 1

  hooks:
    install:
      steps:
        - name: deploy
          # any container image
          image: alpine/helm:3.14
          args:
            - helm
            - install
            - my-app
            - ./chart

  actions:
    - name: restart
      steps: [...]
Hooks are container steps — Helm, kubectl, Ansible, bash. Any image works.

Custom Actions

Beyond Install / Upgrade / Remove

actions:
  - name: backup
    displayName: "Create Backup"
    parameters:
      - name: backupName
        type: string
        required: true
      - name: compress
        type: bool
        default: true
    steps:
      - name: backup
        image: restic/restic
        args:
          - backup
          - /data
          - --tag
          - "{{backupName}}"

Action flow

User triggers action
Action CR created
Argo Workflow executes
Status reported ✓
Examples: backup/restore, user import, cache flush, config reload, DB migration, scaling…

Cozystack → Application Platform

Five principles that transform a PaaS into a full application platform.

01
Universal Abstraction
One template for any app — Helm, Ansible, custom scripts. No new CRD per service.

Extensible
